       - cursorPosition이 제공되면 해당 위치가 visible window 안에 오도록 window offset 계산

         b. 커서 위치 기준으로 window를 결정: 커서가 window 왼쪽 밖이면 window를 왼쪽으로, 오른쪽 밖이면 오른쪽으로 이동
         c. CJK 경계 처리: 2-column 문자가 window 시작/끝에 걸리면 제외

       - getVisibleWindow: 긴 ASCII 텍스트 + cursor 중간 위치

       - `cursorPosition` useState도 추가 (렌더링 트리거용)

    2. useInput 핸들러 수정:
       a. **좌 화살표** (`key.leftArrow`): cursorPositionRef를 Math.max(0, current - 1)로 이동. Array.from(text) 기준 문자 단위.
       b. **우 화살표** (`key.rightArrow`): cursorPositionRef를 Math.min(chars.length, current + 1)로 이동.
       c. **문자 입력** (기존 append): Array.from(textRef.current)로 분해 → splice(cursorPosition, 0, input) → join. cursorPosition += Array.from(input).length.
       d. **백스페이스** (기존 pop): cursorPosition > 0일 때만 동작. splice(cursorPosition - 1, 1). cursorPosition -= 1.
       e. **Enter**: cursorPosition을 0으로 리셋 (기존 텍스트 클리어와 함께).
       f. **히스토리 (up/down)**: 히스토리 항목 로드 시 cursorPosition = 새 텍스트의 chars.length (끝으로 이동).

    3. 렌더링 수정:
       a. `getVisibleWindow(text, availableWidth, cursorPosition)` 호출로 교체
       b. visibleText 렌더링 + cursorOffset 위치에 커서(▏) 표시
       c. 커서 렌더링: visibleText를 cursorOffset 기준으로 분할하여 `<Text>{before}</Text><Text color={theme.text.primary}>▏</Text><Text>{after}</Text>` 패턴
       d. stringWidth 기반으로 before/after 분할 (`.length` 사용 금지)

    4. IMETextInput props 확장:
       a. `onTextChange?: (text: string) => void` — 텍스트 변경 시 콜백 (Plan 02 자동완성용)
       b. 텍스트가 변경될 때마다 (문자 입력, 백스페이스, 히스토리) onTextChange 호출
