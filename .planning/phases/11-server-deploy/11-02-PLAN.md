  <action>
    **Dockerfile** (multi-stage build, Node.js 20 LTS):

    ```dockerfile
    # Stage 1: Build
    FROM node:20-slim AS builder
    WORKDIR /app
    COPY package.json package-lock.json ./
    COPY packages/shared/package.json packages/shared/
    COPY packages/server/package.json packages/server/
    # client package.json needed for workspace resolution but not installed
    COPY packages/client/package.json packages/client/
    RUN npm ci --workspace=packages/server --workspace=packages/shared --include-workspace-root
    COPY packages/shared/ packages/shared/
    COPY packages/server/ packages/server/
    RUN npm -w packages/server run build

    # Stage 2: Production
    FROM node:20-slim
    WORKDIR /app
    ENV NODE_ENV=production
    COPY --from=builder /app/packages/server/dist ./dist
    COPY --from=builder /app/packages/server/package.json ./
    COPY --from=builder /app/node_modules ./node_modules
    EXPOSE 3456
    CMD ["node", "dist/index.js"]
    ```

    주의사항:
    - server의 tsdown build가 shared를 인라인하므로 dist/index.js만 있으면 됨
    - `process.env['PORT']`를 서버가 이미 읽으므로 Fly.io PORT 환경변수 자동 호환
    - geoip-lite는 runtime dependency이므로 node_modules 필요

    **fly.toml**:

    ```toml
    app = 'hivechat'
    primary_region = 'nrt'

    [build]

    [http_service]
      internal_port = 3456
      force_https = true
      auto_stop_machines = 'stop'
      auto_start_machines = true
      min_machines_running = 0

    [[vm]]
      size = 'shared-cpu-1x'
      memory = '256mb'
    ```

    주의: WebSocket은 Fly.io HTTP service에서 자동 지원 (별도 설정 불필요).
    auto_stop_machines = 'stop'으로 유휴 시 비용 절감.

    **.dockerignore**:

    ```
    node_modules
    .git
    .planning
    .claude
    docs
    *.md
    packages/client
    .env*
    ```

    packages/client를 제외하여 Docker 빌드 최적화 (서버 배포에 불필요).
    단, packages/client/package.json은 Dockerfile에서 명시적 COPY하므로 문제없음.
