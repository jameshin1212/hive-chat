프로젝트 루트에 README.md를 작성한다. 아래 섹션 구조를 따를 것:

1. **헤더**: HiveChat 로고 텍스트 + 한 줄 설명 ("Chat with nearby developers right from your terminal")
   - npm version badge: `[![npm](https://img.shields.io/npm/v/hivechat)](https://www.npmjs.com/package/hivechat)`
   - license badge: `[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](LICENSE)`
   - Node.js badge: `[![Node.js](https://img.shields.io/badge/node-%3E%3D20-brightgreen)](https://nodejs.org)`

2. **스크린샷 섹션**: `<!-- screenshots -->` 주석으로 placeholder 영역 생성. 아래 형태로:
   ```
   <p align="center">
     <img src="docs/screenshot-lobby.png" alt="Lobby - nearby users" width="600">
   </p>
   ```
   lobby, chat, friends 3개 placeholder 이미지를 넣되, 이미지가 없으면 보이지 않도록 주석 처리. 사용자가 나중에 실제 스크린샷으로 교체할 수 있게 `docs/` 디렉토리 경로 사용.

3. **Features 섹션**:
   - Nearby Discovery: IP geolocation 기반 근처 사용자 발견 (1/3/5/10km 범위)
   - 1:1 Chat: 실시간 채팅 (relay + P2P direct connection)
   - Friends: nick#tag로 친구 추가/관리
   - P2P: Hyperswarm 기반 NAT traversal, relay fallback
   - Korean IME: 한글 입력 완벽 지원
   - TUI: Ink + React 기반 터미널 UI

4. **Quick Start 섹션**:
   ```bash
   npx hivechat
   ```
   - Node.js 20+ 필요
   - 자동으로 wss://hivechat-signal.fly.dev 서버에 연결
   - 첫 실행 시 닉네임 설정 + AI CLI 선택

5. **Commands 섹션**: 슬래시 명령어 테이블
   | Command | Description |
   - /chat nick#tag - 1:1 채팅 시작
   - /add nick#tag - 친구 추가
   - /remove nick#tag - 친구 삭제
   - /friends - 친구 목록
   - /settings - 설정 변경
   - /back - 로비로 돌아가기
   - /exit - 종료

6. **How It Works 섹션**: 간단한 아키텍처 설명
   - Signal server (Fly.io): 사용자 발견 + relay fallback
   - P2P (Hyperswarm): 직접 연결 성공 시 서버 거치지 않음
   - 메시지는 저장되지 않음 (ephemeral)

7. **Tech Stack 섹션**: 주요 기술 목록 (간단한 테이블)

8. **Development 섹션**: 로컬 개발 방법
   ```bash
   git clone https://github.com/jamie-chat/hivechat.git
   cd hivechat
   npm install
   npm run dev    # 또는 해당 명령어
   npm test
   ```

9. **License**: MIT
