**AsciiBanner.tsx 수정:**

1. `Breakpoint` 타입을 `useTerminalSize.ts`에서 import
2. `AsciiBannerProps` interface 추가: `{ breakpoint?: Breakpoint }`
3. breakpoint에 따른 렌더링 분기:
   - `'standard'` 또는 `'wide'` (또는 prop 미전달 시 기본값): 기존 figlet 'Standard' 폰트 배너 표시
   - `'compact'`: plain text fallback 표시. 내용은 `'=== HIVECHAT ==='` (기존 catch fallback과 동일)
4. figlet 배너는 기존처럼 모듈 로드 시점에 한 번만 생성 (성능)
5. 기존 export signature를 유지하되 optional prop 추가 (backward compatible)

**OnboardingScreen.tsx 수정:**

1. `useTerminalSize` hook import 추가
2. step 타입 변경: `'welcome' | 'nickname' | 'ai-cli'` → `'nickname' | 'ai-cli'` (welcome step 제거)
3. 초기 step을 `'nickname'`으로 설정 (welcome 제거로 바로 닉네임 입력)
4. `useTerminalSize()` 호출하여 `breakpoint` 획득

5. **StepIndicator 인라인 렌더링** (별도 컴포넌트 불필요):
   - 닉네임 step: `<Text dimColor>Step 1/2</Text>`
   - AI CLI step: `<Text dimColor>Step 2/2</Text>`
   - 배너 바로 아래에 배치

6. **Box border 적용** (user decision: single border, cyan):
   - 닉네임 입력 영역을 `<Box borderStyle="single" borderColor="cyan" flexDirection="column" paddingX={1}>` 로 감싸기
   - AI CLI 선택 영역도 동일한 Box border로 감싸기
   - Box 내부에 label 텍스트 ("Choose a nickname", "Select your AI CLI tool")와 입력/선택 컴포넌트 배치

7. **AsciiBanner에 breakpoint prop 전달**: `<AsciiBanner breakpoint={breakpoint} />`

8. **welcome step의 'Press Enter to continue' 제거** — 첫 화면이 바로 닉네임 입력 + 배너

9. 기존 error 표시 로직 유지 (theme.text.error 색상)

10. IMETextInput의 isActive는 step === 'nickname'일 때만 true (이미 AiCliSelector에 isActive 있으므로 키 충돌 방지). IMETextInput에 isActive prop이 있는지 확인하고, 없으면 step 분기 렌더링으로 충분 (현재 이미 조건부 렌더링).

**borderColor 값**: `theme.text.info`가 아닌 직접 `"cyan"` 문자열 사용도 가능하나, 일관성을 위해 `theme.text.info` 사용. Ink의 `borderColor`는 string 타입이므로 theme 값 직접 전달 가능.
