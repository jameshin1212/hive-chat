    **2. Create useChatSession hook** at `packages/client/src/hooks/useChatSession.ts`:

    Follow `useNearbyUsers` pattern — subscribe to SignalingClient events, manage React state.

    ```typescript
    import { useState, useEffect, useCallback, useRef } from 'react';
    import type { SignalingClient } from '../network/SignalingClient.js';
    import type { NearbyUser, ChatSessionStatus, ChatMessage, Identity } from '@cling-talk/shared';
    import { CHAT_REQUEST_TIMEOUT_MS, MAX_MESSAGES } from '@cling-talk/shared';

    interface IncomingRequest {
      sessionId: string;
      from: NearbyUser;
    }

    interface ChatSessionReturn {
      chatStatus: ChatSessionStatus;
      partner: NearbyUser | null;
      sessionId: string | null;
      chatMessages: ChatMessage[];
      incomingRequest: IncomingRequest | null;
      requestChat: (target: NearbyUser) => void;
      acceptRequest: () => void;
      declineRequest: () => void;
      sendMessage: (content: string, myIdentity: Identity) => void;
      leaveChat: () => void;
    }

    export function useChatSession(client: SignalingClient | null, connectionStatus: string): ChatSessionReturn
    ```

    State:
    - `chatStatus: ChatSessionStatus` — starts 'idle'
    - `partner: NearbyUser | null` — current chat partner
    - `sessionId: string | null` — active session ID
    - `chatMessages: ChatMessage[]` — in-memory only, capped at MAX_MESSAGES
    - `incomingRequest: IncomingRequest | null` — pending incoming request

    Event handlers (subscribe in useEffect, cleanup on unmount):
    - `chat_requested`: Set `incomingRequest` state. If already in a chat (chatStatus === 'active'), auto-decline.
    - `chat_accepted`: Set chatStatus='active', partner from event data, sessionId. Clear any request timeout.
    - `chat_declined`: Set chatStatus='idle'. Add system message "No response from user".
    - `chat_msg`: Append to chatMessages (cap at MAX_MESSAGES). Ring terminal bell: `process.stdout.write('\x07')`. Generate message ID with `crypto.randomUUID()`.
    - `chat_left`: Add system message "[nick#tag] left the chat". Set partner to null but keep chatStatus='active' so user can /leave themselves. Actually per CONTEXT: keep chat screen open. Set a flag like `partnerLeft: true`.
    - `chat_user_offline`: Add system message "[nick#tag] went offline". Keep chat screen open (per CONTEXT decision).
    - `chat_error`: If code is 'USER_OFFLINE' or 'USER_BUSY', add system message and set chatStatus='idle'.

    Connection status integration:
    - When `connectionStatus` changes to 'reconnecting', set chatStatus to 'disconnected' (but preserve partner and messages).
    - When `connectionStatus` changes back to 'connected' AND there was a previous partner, automatically re-request chat: `client.requestChat(partner.nickname, partner.tag)`, set chatStatus to 'requesting'. Per RESEARCH.md Pitfall 3.

    Actions:
    - `requestChat(target)`: Call `client.requestChat(target.nickname, target.tag)`. Set chatStatus='requesting', partner=target. Start 30s timeout — if no response, set chatStatus='idle' and add system message "No response".
    - `acceptRequest()`: Call `client.acceptChat(incomingRequest.sessionId)`. Set chatStatus='active', partner from incomingRequest.from, sessionId. Clear incomingRequest.
    - `declineRequest()`: Call `client.declineChat(incomingRequest.sessionId)`. Clear incomingRequest. (Silent — per CONTEXT decision.)
    - `sendMessage(content, myIdentity)`: Call `client.sendChatMessage(sessionId, content)`. Add message to chatMessages locally with myIdentity as `from`, `Date.now()` as timestamp, `crypto.randomUUID()` as id. Do NOT ring bell for own messages.
    - `leaveChat()`: Call `client.leaveChat(sessionId)`. Reset chatStatus='idle', partner=null, sessionId=null, chatMessages=[].
